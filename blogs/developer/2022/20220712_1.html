    GitHub 清空仓库 | reset 仓库

    osoke 2022/7/12 GitHubgit

    # 场景

    想象这样一个情况，你用 Github Pages 搭建博客， 发现博客内容有点乱，你想清空 github 仓库重新来过，注意不是删除仓库，而是将仓库 reset 一下，回到刚创建好仓库，你正要做第 1 次 commit 的时候。可以实现这种操作吗？可以

    # 原理

    首先准备好你第一次提交的文件，现在你的仓库有1个默认 分支 A, 你可以创建1个新的 分支 B, 将准备提交的文件 commit 到 分支B 中。此时就可以删除默认 分支 A, 然后将 分支 B 设为默认分支，到这里，仓库已经在本地被 reset, 最后 push 分支 B 即可

    具体命令如下：

    - 克隆并查看当前分支： main/master
git clone https://github.com/U/U.github.io.git
git branch

将准备提交的文件提交到新分支
- 清空当前仓库
- 将准备提交的文件copy进来
git checkout --orphan newBranch  # 生成 新分支
git add .
git commit -m "init"    # commit 之后新分支才会被创建

git branch -D main      # 删除-默认分支
git branch -m main      # 将当前新分支设为-默认分支

git push -f origin main # 强制推送新分支
