    MarginNote to Markdown

    osoke 2022/7/25 知识管理MarginNoteObsidianMarkdownpandoc

    # 场景

    在 iPad 上使用 MarginNote 看 PDF, 添加摘录、笔记, 然后将 MarginNote 中的笔记导出到 Obsidian 中统一管理, 方便查找。 以下为尝试过的方法:

    # 方法1: 先转「思维导图」再转 md

    1. MarginNote 中将笔记本导出为 MindManager 和 iThoughts
    2. 使用 iThoughts 导入上一步导出的 .mmap 文件
    3. iThoughts 右上分享 ▶ 打开方式 ▶ Markdown ▶ (.textpack)

    优点: 最后导出的文件类似于「大纲笔记」, 保留了原内容的层次结构, 只需稍加整理即可使用。

    # 方法2: 先转 word 再转 md

    1. 将笔记本导出为 MS Word(docx)
    2. 使用 pandoc 将 word 转为 md
      • 首先下载、安装 pandoc (opens new window)
      • 打开 CMD 在 「word 文件所在目录」运行以下命令:
    pandoc -f docx -t markdown --extract-media ./images -o dest_文件.md src_文件.docx
    1

    转换之后, word 中的图片会提取到 ./images/media 目录下

    问题: 转换之后的 md 有很多冗余的内容, 清理起来很麻烦

    # 方法3: 使用插件

    同时打开 MN 的 Obsidian-bridge 插件和 Ob 的 MarginNote Companion 插件

    具体使用是这样的: 需要以分屏的方式同时打开 MarginNote 和 Obsidian, 在 MarginNote 做好一个脑图卡片, 选中卡片并粘贴到 Obsidian 中。

    该方法只是大概了解了一下, 无法一次性完整导出整个笔记本, 最多支持一次性导出所有的卡片标题, 想要完整导出就得逐个卡片导出, 不太方便, pass。

    # 选哪个方法?

    推荐方法1, 但是图片有个小问题

    • iThoughts 导出 markdown·textpack 默认带图片
    • 但是图片是「注释+引用」的形式, 在 Obsidian 中无法显示 (Typora 可以显示)

    写个 python 脚本替换一下

    import re  
  
# 解压目录  
base = input('textpack·text.md 文件路径:\n').strip('"')  
  
# 将 text.md 中的图片引用 ![][1] 修改为 ![[assets/abc.png]] 格式  
def replacetext():  
    with open(base, 'r+', encoding='utf-8') as f:  
        data = f.read()  
        pics = re.findall('assets/.*?.png', data)  # 文件末尾有所有 [图片] 注释  
  
        for pic in pics:  
            data = re.sub('\[\]\[\d+\]', f'[[{pic}]]', data, count=1)  # 每次替换1个  
  
        f.seek(0)  
        f.write(data) # 从 0 插入 new data
        f.truncate()  # 去除  old data  
  
replacetext()  
print('\n 图片链接修改成功 🎉🎉🎉\n')

    # 具体导入步骤

    1. 将导出的 .textpack 重命名为 .zip 并解压
    2. 运行上述 .py 脚本, 输入上述解压目录中 text.md 文件全路径, 进行转换
    3. 最后将 text.md 和 assets 文件夹 拷贝到 obsidian 中即可
